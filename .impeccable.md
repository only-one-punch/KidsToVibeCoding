# Design Context

## Users

**Primary**: 6-14岁儿童，编程零基础或初学者
- **6-9岁**: 需要图形化、大按钮、简单交互
- **10-14岁**: 可以处理更复杂的界面和概念

**Secondary**: 家长
- 关注学习进度和安全
- 需要清晰的报告和控制面板

**Context**: 孩子们在家里、课后或周末使用，可能是独立学习或与家长一起。目标是让学习 AI 工具像玩游戏一样有趣。

## Brand Personality

**3 words**: 友好 · 游戏化 · 专业

**Tone**: 像一个耐心的朋友，不是严厉的老师。鼓励探索、允许犯错、庆祝每一个小成就。

**Emotional goals**:
- 自信：我能用 AI 创造东西
- 好奇：接下来会发生什么？
- 成就感：看，这是我做的！

## Aesthetic Direction

**Visual tone**: 明亮轻快，但不过于幼稚。专业感 + 亲和力。

**Color palette**:
```
Primary: 珊瑚橙 #FF6B6B (温暖、活力)
Accent:  青绿 #2DD4BF (清新、成长)
Highlight: 金黄 #FBBF24 (成就、奖励)
Neutral: 暖色调的灰 (不使用纯灰色)
Background: 奶油色 #FFF9F5 (温暖、不刺眼)
```

**Typography**:
- Headlines: Outfit (bold, friendly)
- Body: Noto Sans SC (清晰易读)
- Code: JetBrains Mono

**References**:
- Duolingo: 游戏化、成就系统、可爱但不幼稚
- Google Stitch: 现代感、大留白、柔和渐变

**Anti-references**:
- 不要紫色渐变背景
- 不要居中对齐的所有内容
- 不要相同大小的卡片网格
- 不要在彩色背景上放灰色文字

## Design Principles

1. **0 编程友好**: 每个界面都要让零基础的孩子能理解
2. **即时反馈**: 点击就有反应，让用户知道发生了什么
3. **游戏化驱动**: 积分、徽章、进度条，但不强制
4. **温暖不幼稚**: 专业的设计语言，但用温暖的色彩和圆润的形状
5. **避免 AI slop**: 不用紫色渐变、相同卡片网格、过度居中

## Technical Constraints

- **Accessibility**: WCAG AA 标准，清晰的焦点状态
- **Responsive**: Mobile-first，最小点击区域 44x44px
- **Motion**: 尊重 prefers-reduced-motion
- **Performance**: 快速加载，流畅动画 (60fps)

## Components

- 大按钮、大图标 (最小 44px)
- 圆角卡片 (radius: 12-24px)
- 清晰的层级和间距
- 有意义的动画 (状态变化、引导注意)
