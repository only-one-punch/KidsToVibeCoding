## 1. 产品概述

### 1.1 产品定位
****
CodeBuddyAI 是一个面向 6-14 岁儿童的 AI 驱动编程综合学习平台，通过数字人虚拟老师、多模态交互、实时可视化等创新技术，降低编程学习门槛，让儿童在趣味化、个性化的环境中掌握编程技能。

**核心价值主张**:
- **降低门槛**: 语音交互 + 画板输入，无需熟练打字
- **可视化学习**: 代码执行过程"看得见"，抽象概念具象化
- **个性化陪伴**: AI 老师记住每个孩子的学习进度和风格
- **趣味化体验**: 游戏化元素 + 成就系统，保持学习动力

### 1.2 目标用户

**主要用户**: 6-14 岁儿童
- **初级用户 (6-9岁)**: 编程零基础，需要图形化编程入门
- **用户 (10-14岁)**: 有一定基础，希望学习文本编程

**次要用户**: 家长
- 关注孩子学习进度和效果
- 需要学习报告和数据反馈
- 可能需要控制使用时长

### 1.3 市场分析

**竞品对比**:
| 产品 | 优势 | 劣势 | 差异化 |
|------|------|------|--------|
| Scratch | 图形化编程，社区活跃 | 缺乏AI辅导，无个性化 | 我们有AI老师实时指导 |
| Code.org | 课程体系完善 | 交互单一，无语音输入 | 我们支持多模态交互 |
| 编程猫 | 本土化，游戏化 | AI能力弱，可视化有限 | 我们有更强的AI和可视化 |

**市场机会**:
- 少儿编程市场规模持续增长
- AI技术成熟，可以提供更好的学习体验
- 家长对STEM教育的重视度提升

### 2.2 核心用户场景

#### 场景1: 新用户首次学习
**用户**: 7岁儿童，编程零基础
**目标**: 完成第一节编程课
**流程**:
1. 注册账号，选择年龄段和兴趣
2. 选择喜欢的数字人老师形象
3. 观看欢迎动画，了解平台功能
4. 进入第一节课"认识代码"
5. 数字人老师讲解基础概念
6. 完成简单的拖拽编程任务
7. 获得第一个成就徽章

**关键体验**:
- 界面简洁，大图标大按钮
- 数字人老师亲切友好
- 任务简单，容易完成
- 即时反馈和鼓励

#### 场景2: 语音提问
**用户**: 9岁儿童，遇到代码错误
**目标**: 理解错误原因并修复
**流程**:
1. 代码运行报错，显示错误信息
2. 孩子不理解，点击麦克风按钮
3. 说话: "这是什么意思？为什么错了？"
4. 语音识别，显示文字
5. AI分析代码和错误，生成解释
6. 数字人老师用儿童友好的语言讲解
7. 提供修改建议，孩子修改代码
8. 代码运行成功，数字人庆祝

**关键体验**:
- 语音识别准确
- 解释通俗易懂
- 数字人表情配合（困惑→理解→高兴）
- 引导而非直接给答案

#### 场景3: 算法可视化学习
**用户**: 12岁儿童，学习排序算法
**目标**: 理解冒泡排序原理
**流程**:
1. 进入"排序算法"课程
2. 数字人老师讲解冒泡排序概念
3. 播放可视化动画，展示排序过程
4. 孩子可以暂停、回放、单步执行
5. 尝试自己编写冒泡排序代码
6. 运行代码，实时看到可视化效果
7. 对比自己的实现和标准实现
8. 完成课程，获得"排序大师"徽章

**关键体验**:
- 可视化动画流畅直观
- 可以控制播放速度
- 代码和可视化同步
- 成就感强

## 3. 技术架构概述

### 3.1 八大核心技术

1. **数字人**: 虚拟AI老师，提供情感化交互
2. **多模态**: 语音、画板、文字多种输入方式
3. **流式输出**: 实时响应，保持儿童注意力
4. **RAG**: 知识检索，提供准确的编程知识（分层：初级/中级/高级）
5. **记忆系统**: 追踪学习进度，个性化教学，支持难度自适应
6. **数据可视化**: 代码执行可视化，学习报告
7. **MCP**: 工具调用能力（代码执行、可视化生成、文件管理）
8. **Skills**: 标准化教学流程（LangGraph编排）

**儿童友好的Skills体系**（基于AI架构师设计）:
- **教学类**: 友好解释、分步引导、可视化调试、正向激励
- **创作类**: 模板生成、智能提示、创意激发、安全检查
- **学习管理**: 难度自适应、游戏化路径、家长报告

### 3.2 技术栈（已确认）

**前端技术栈**:
- 框架: React 18 + TypeScript
- 构建工具: Vite
- 状态管理: Zustand（确认）
- UI框架: Ant Design + 自定义儿童主题（产品/前端推荐）
- 代码编辑器: Monaco Editor (VS Code同款)
- 数字人渲染: 讯飞数字人接入
- 可视化: Canvas / D3.js / ECharts / three.js
- 实时通信: WebSocket

**后端技术栈**:
- 框架: FastAPI (异步支持)
- AI集成: LangGraph
- 向量数据库: MVP优先 Pinecone（托管，低运维）；长期可评估 Chroma / Qdrant (这里也可以接入一些API接口)
- 关系数据库: PostgreSQL
- 缓存: Redis (短期记忆 + 缓存)
- 异步任务: Celery + Redis
- 代码执行: Docker容器隔离

**AI能力**（基于AI架构师详细方案）:
- LLM主模型: claude opus 4.6（教育场景优先）
- LLM备选: GLM-5（注意这个不是多模态模型）
- RAG: 优先 Pinecone + 分层知识库（初级/中级/高级）
- 记忆系统: Redis（短期会话）+ PostgreSQL（长期学习记录）+ 语义记忆检索
- 难度自适应: 根据年龄（6-8/9-11/12-14）和能力动态调整
- MCP: 工具调用框架（代码执行、可视化、文件管理、文档检索）
- Skills: LangGraph编排，儿童友好的教学流程
- 多模态: Whisper（语音识别）+ Vision模型（画板理解）
- 安全机制: 内容过滤、代码沙箱、家长控制、儿童隐私保护

**部署方案**:
- 云服务: 阿里云
- 容器化: Docker + Kubernetes
- CDN: 静态资源加速
